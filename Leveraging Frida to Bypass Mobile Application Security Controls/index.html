What is Frida

man frida
Frida is an open-source dynamic instrumentation toolkit that allows developers, reverse engineers, and security researchers to monitor, intercept, and modify the behavior of running applications in real-time.

Get Swift Methods

frida -U -l list_swift_methods.js -f com.darkbank.defcon 
if (Swift.available) {
    console.log("[+] Successfully Connected and Detected Swift");
    Process.enumerateModules().forEach(function (mod) {
        const matches = mod.enumerateSymbols().filter(function (symbol) {
            return symbol.name.includes("IOSSecuritySuite");
        });
        if (matches.length > 0) {
            console.log("[*] Found: " + matches.length + " match(es) in: " + mod.name);
            matches.forEach(function (symbol) {
                console.log("Symbol Address: " + symbol.address + " Symbol Name: " + symbol.name);
            });
        }
    });
} else {
    console.log("[-] Swift Not Available");
}

The Joys of Mangling

swift-demangle
Swift name mangling is an internal compiler process that converts human-readable source code names (like functions, classes, and properties) into unique, standardized strings within the compiled binary

Putting the Final Pieces Together

frida -U -l validate.js -f com.darkbank.defcon 
if (Swift.available){
    console.log("[+] Successfully Connected and Detected Swift");
    const mod = Process.getModuleByName("DarkBank");
    const target = mod.enumerateSymbols().find(function (symbol) {
        return symbol.name === "$s16IOSSecuritySuiteAAC13amIJailbrokenSbyFZ";
    });
    if (!target) {
        console.log("[-] Symbol not found");
    } else {
        console.log("[+] Hooking amIJailbroken()");
        Interceptor.attach(target.address, {
            onEnter: function (retval) {
                console.log("Boolean Value for amIJailborken() Set to: " + Boolean(retval))
            }
        });
    }
}
else{
    console.log("[-] Swift Not Available")
}

Key Takeaways

takeaways.md
  • Frida enables dynamic instrumentation, allowing security testers to observe and modify an application's behavior in real time during a mobile penetration test.
  • Systematic enumeration of Objective-C classes, Swift symbols, and application methods is essential for understanding how an iOS application's security mechanisms are implemented.
  • Understanding Swift name mangling and using tools like swift-demangle helps map compiled symbols back to meaningful functions for analysis.
  • By identifying and hooking the correct Swift function, Frida can intercept runtime execution and alter application logic, such as changing a jailbreak detection result.
  • A structured workflow of enumeration, symbol discovery, analysis, and runtime hooking provides a repeatable methodology for assessing and testing mobile application security controls.
